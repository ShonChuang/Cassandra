# CAP 定理

  • Ｃ 代表一致性(Consistency)
  • Ａ 代表可用性(Availability)
  • Ｐ 代表分區容錯性(Parttion tolerance)

該定理認為只能滿足當中兩項，無法三項全部完成

# 犧牲一致性

該系統稱AP系統，採用的是最終一致性的方式(Cassandra 認為可用性和分區容錯性比一致性更重要，所以採用此系統)

# 犧牲可用性

該系統稱CP系統，在資料未同步前拒絕回應，通常在分散式交易中

# 犧牲分區容錯性

在分散式系統中該系統不存在，因無法擴展。

# 調整一致性-寫入

# ANY

任意節點可寫入則操作成功，除非全部節點都掛了。「最低的Ｃ，最高到A」

# ALL

必須所有的節點都寫入才操作成功。「最高C，最低A」

# EACH_QUORUM

必須寫入每個數據中心的quorum(過半數)的數量的副本節點寫入成功，才操作成功。

# LOCAL_ONE

任何數據中心的副本節點寫入則操作成功。

# LOCAL_QUORUM

本定本定數據中心的quorum(過半數)的數量的副本節點成功寫入則操作成功。可避免跨區域數據中心。

# LOCAL_SERIAL

本定本定數據中心的quorum(過半數)的數量的副本節點，且有條件的，且有條件的寫入成功，才操作成功。

# ONE

任一個副本操作寫入則操作成功。最近的節點才則優先處理。

# 讀取的一致性分類

# ALL

所有節點副本都要回應才讀取成功。

# EACH_QUORUM

向每個數據中心的quorum(過半數)的數量的副本節點讀取都讀取都返回最新時間，才操作成功。

# SERIAL

讀取當前的數據，包含未認可的數據。

# LOCAL_SERIAL

限定讀取local的數據，包含未認可的數據。

# QUORUM

讀取每個數據中心的quorum(過半數)的數量的副本節點返回，才操作成功。

# LOCAL_QUORUM

讀取每個Local數據中心的quorum(過半數)的數量的副本節點返回，才操作成功。可避免跨區域數據中心。

# ONE

讀取由snitch決定的最近的副本返回結果。

# LOCAL_ONE

返回Local最近的副本返回結果。

# TWO

返回最近兩個數據。

# THREE

返回最近三個數據。

quorum(過半數) 計算方式（四捨五入）：

quorum = (副本數量 / 2)+1

如：

  • replication factor是3，quorum值為2-表示集群可以最多容忍1個節點掛掉

  • replication factor是6，quorum值為4-表示集群可以最多容忍2各節點掛掉。

要確保讀寫一致性建議的公式為：

(nodes_written + nodes_read) > replication_factor

策略：

nodes_written nodes_read replication_factor consistency
ONE QUORUM 3 false
ONE ALL 3 true
ONE ONE 3 false
QUORUM QUORUM 3 true
